场景黑屏问题（屏幕黑但对话继续显示）

现象
一些场景过渡后画面变黑，但对话文字和选择框仍然正常显示在黑色背景上。

根因分析

1. ScreenOverlay(Black) 永不自动消失
rendering.rs:926-946 — update_overlay_tween 在 tween 完成后：
if tween.end_alpha <= 0.0 {
    commands.entity(entity).insert(Visibility::Hidden);
}
end_alpha = 1.0（ScreenOverlay 淡入到全黑）时不满足条件，overlay 永远留在画面上。
对话显示在 overlay 之上（dialogue.rs ZIndex 3 vs overlay ZIndex 65534），所以黑屏上仍可见。

2. 条件跳转不清理场景（主要触发路径）
script_runner.rs:355-372（skip）和 864-882（normal）
Condition / AffectionCondition / Exif 跳转标签时不调用 clear_scene_sprites()。
而 Jump / Call / CallScript 是调用的。

常见触发脚本模式：
ScreenOverlay(Black, 500)         // 场景开始，淡入黑
Condition { goto: "somewhere" }  // 条件满足，跳转
// → overlay 卡黑，目标标签的对话正常显示

3. clear_scene_sprites() 只清理 SpriteOverlay
script_runner.rs:1589-1602 — Jump/Call 虽然调用 clear_scene_sprites()，但它只清
SpriteOverlayManager.sprites（DrawSprite），不碰 ScreenOverlayRoot、CgRoot、背景。

触发场景
a) ScreenOverlay + 条件跳转之间没有 ClearOverlay（最常见）
b) 场景间 Jump 前未执行的 ClearOverlay
c) ShowCG 后 SetBg 自动销毁 CG（rendering.rs:227-235），但对话系统继续
