Character Sprite Architecture

How should sprites be rendered and managed?

Sprite positions on 1280x720 (sprites are 780x720)
Left (x: 0)
Center (x: 156)
Right (x: 312)
Dialogue box (200px → scaled to 125px in mockup)
Sprites overlap bg, dialogue box rendered on top

Approach

A

Per-character entities

ShowFg → spawn sprite entity + CharacterSprite component. HideFg → despawn. Clean ECS, easy transitions per entity.

Simple, testable, natural ECS
Need to track which chars are visible
B

Pooled sprite entities

Pre-spawn N sprite entities (e.g. 3 for L/C/R). ShowFg assigns texture + position. Reuses entities, fewer spawn/despawn.

No alloc churn, good perf
More complex state mgmt

Texture loading

A

On-demand (lazy)

Load texture via AssetServer when ShowFg fires. Simpler startup, slight frame delay on first show.

B

Preload all

Load all FG textures at Gameplay OnEnter. Instant transitions, longer initial load.

C

Cache-on-use

Load on first use, cache handle in resource. Subsequent ShowFg instant. Best of both.
